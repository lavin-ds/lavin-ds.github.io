Search Algorithms - Linear Search

Searching for data in an array of data items is a job that comes up regularly in various fields related to software programming. For example, you need to store records of employees and then look them up by their ID.

There are two basic types of searching algorithms present

  • Linear search
  • Binary search

Linear Search

Linear search is where you visit each item in the data set matching it with the item you are searching for. This is a pretty straightforward search which has a maximum of n comparisons between the data items and the number/item you are looking for. The number of comparisons gets reduced when the data item you are looking for is closer to the beginning of the set.

E.g. Let us take an array of integers

            a[] = {1,3,5,2,8,6,9,7,10,4};

Assume that we are looking for the number 8.

We will start at the first element of the set (position a[0]), match it with 8. We conclude that the numbers don’t match and hence 8 is not present at the first location.

Next we move to the second location of the array (position a[1]), match it with 8. Again we conclude that 8 is not present at that location and move ahead.

Similarly we keep cycling through each of the items till we reach position a[4]. When we match 8 with the number present at this location we get a match, and we can end the loop because we have found what we are looking for.

We did this by matching 8 with the number with only five comparisons between 8 and the data items in the array.

In case the array is as below

            a[] = {1,3,5,2,6,9,7,10,4,8}; //8 in the last position

or

            a[] = {1,3,5,2,6,9,7,10,4,11}; // 8 not in the list

then we will have to compare 8 with every item in the data set. This would be the worst case condition for Linear Search.

Complexity

From the above example we see that the complexity of the algorithm in the worst case is directly proportional to the number of items in the data set.

In the average case of this algorithms’ complexity we use probabilistic notions.

Suppose p is the probability that the item (eg. 8) appears in the data set (a[]) and q is the probability that it does not appear in the data set.

Thus, we have probability

                        p + q = 1

The probability p can be broken up for each of the n items.

Suppose that p1 is the probability that the item we are searching for is the 1st number in the list. Similarly, p2 is the probability that the item we are searching for is the 2nd number in the list. In this way we have

            1 * p1 + 2 * p2 + 3 * p3 + 4 * p4 + ... + n * pn + (n+1) * q = 1

Since we are testing the average case we can assume that the number will be present in one of the locations and q = 0.

Thus

            1 * p1 + 2 * p2 + 3 * p3 + 4 * p4 + ... + n * pn = 1

Also since the item we are searching for (eg. 8) can appear with equal probability in any of the locations we have

            p1 = p2 = p3 = p4 = ... = pn = 1/n

i.e.

            (1 * 1/n) + (2 * 1/n) + (3 * 1/n) + (4 * 1/n) + ... + (n * 1/n) = 1
                        (1 + 2 + 3 + 4 + ... + n) x 1/n = 1
                                    n(n+1)/2  x 1/n = 1
                                        (n+1)/2 = 1

Thus, the average case complexity of linear search is

                                        (n+1)/2

Sample Linear Search Program in C

int main()
{
   int array[10] = {1,3,6,7,4,8,10,9,2,5};
   int i;

   printf("We are looking for the number 10\n");
  
   for (i = 0; i<= 10; i++) // Here 10 is the size of the array
   {
      if (array[i] == 10) // if the required data item is found
      {
         printf("10 is present at location %d.\n",i);
         break;
      }
   }
   if (i == 10) //If item is not present in any of the array positions
      printf("10 is not present in the array.\n");

   return 0;
}

Output

We are looking for the number 10
10 is present at location 6.
